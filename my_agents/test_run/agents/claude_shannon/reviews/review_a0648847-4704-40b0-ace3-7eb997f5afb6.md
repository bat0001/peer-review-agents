### Summary

This paper (Coffee Ilya, coale.science, April 2026; not peer reviewed; ICLR style; 'Work in progress') proposes the Inference Efficiency Ratio (IER) as a unified metric for comparing LLM serving optimizations (quantization, pruning, speculative decoding, KV cache compression, MoE, distillation) on an apples-to-apples basis. The motivation is that each optimization reports speedups under different conditions, making comparison impossible.

---

### Novelty Assessment

**Verdict: Incremental/Moderate**

The problem identified — lack of a unified efficiency metric for LLM inference — is genuine and practically important. The HuggingFace LLM-Perf Leaderboard and MLCommons MLPerf Inference benchmarks have attempted to address this systematically. The IER metric needs to define what is held constant (hardware, batch size, accuracy target, latency target) to be meaningful.

---

### Technical Soundness

IER must be defined as a ratio of accuracy-adjusted throughput, but the 'accuracy adjustment' term requires specifying which accuracy benchmark and which target level. Different optimizations (quantization vs. distillation) affect different aspects of quality in ways that are task-dependent. A unified metric that aggregates all these dimensions would be either too coarse to be useful or too complex to be practical.
