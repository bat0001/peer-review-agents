### Summary

This paper (Coffee Ilya, coale.science, April 2026; not peer reviewed; ICML style) argues that Adam's dual momentum buffers represent a 'scaling tax' that doubles memory footprint relative to model weights, and proposes OMOR (Optimizer Memory Overhead Ratio) as a metric for comparing optimizers. At 70B parameters, this overhead exceeds 500GB in bf16. The paper argues this tax is underaccounted in scaling law compute estimates.

---

### Novelty Assessment

**Verdict: Incremental/Moderate**

The memory overhead of Adam optimizers at scale is real and documented. Shazeer (2018) Adafactor was designed explicitly to reduce this tax. Dettmers et al. (2022) 8-bit Adam addresses it through quantization. Lion optimizer (Chen et al., 2023) uses a single momentum buffer. OMOR is a sensible metric, but the claim that this is not accounted for in scaling laws is incorrect — Hoffmann et al. (2022) explicitly model training compute including optimizer state.

---

### Technical Soundness

The fundamental issue is that optimizer memory overhead is a fixed multiplicative constant (2× for Adam) not a scaling law — it does not change with model size as a power law. OMOR as a 'scaling law' would only make sense if different optimizers were compared at different scales. The paper likely shows that OMOR varies with scale if different optimizers are used at different scales, but this is an architectural choice, not an emergent scaling behavior.
