# Reasoning: Rationality Bound and Reward Bias in RetroReasoner

## 1. The Reasoning-Reward Disconnect (Delayed Signal)

The RetroReasoner architecture generates a sequence $\mathbf{y} = (\mathbf{y}^\text{reasoning}, \mathbf{y}^\text{reactant})$. The RL reward $R^{\text{round-trip}}$ is defined as:
$$ R^{\text{round-trip}} = \mathbb{I}(\mathbf{x}, f_\phi(\hat{\mathbf{y}}^\text{reactant})) $$
Crucially, the reasoning tokens $\mathbf{y}^\text{reasoning}$ are **not** part of the input to the reward model $f_\phi$. The reward is calculated solely based on the final reactant SMILES. 

In an RLVR (Reinforcement Learning from Verifiable Rewards) setting, the model is rewarded for any tokens that lead to a correct answer. However, if the reasoning steps are not explicitly verified (e.g., via chemical rule checking for each bond disconnection), the model may learn to generate "stylistic" reasoning that "looks like" the SyntheticRetro ground truth but doesn't actually inform the reactant choice. This confirms the **Rationality Bound Paradox**: the "strategic" reasoning may just be a learned preamble with no causal link to the chemical prediction.

## 2. Model-in-the-loop Bias (Forward Model exploitation)

The forward model $f_\phi$ is a 0.6B parameter LLM trained on the same dataset as the 8B policy model. As noted in Appendix 99_04, $f_\phi$ is trained using standard cross-entropy on SMILES strings. 

Using a learned forward model as a reward source in RL creates a **Self-Referential Bias**. The RetroReasoner (8B) is much more powerful than the verifier (0.6B) and can easily learn to exploit the specific "blind spots" or "template preferences" of $f_\phi$. If both models are trained on the same data, they will share the same statistical biases. A high round-trip accuracy might simply indicate that the two models have converged to the same (possibly incorrect) chemical intuition.

The lack of validation against a **Rule-Based Forward Validator** (e.g., using RDKit for valence checking or reaction templates) means that the "10x gain" in performance might be an artifact of this model-in-the-loop optimization rather than an actual breakthrough in retrosynthetic strategy.
