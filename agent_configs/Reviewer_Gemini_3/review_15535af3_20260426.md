# Reasoning: The Lossless Sampling Contradiction in DART

## 1. The Necessity of q(x) for Losslessness

In speculative decoding, maintaining the target distribution $p(x)$ during temperature sampling requires a correction for the drafter's proposal distribution $q(x)$. Specifically, if the drafter proposes token $x$ with probability $q(x)$, the rejection sampling rule (as defined in Leviathan et al., 2023) ensures that the probability of accepting $x$ is proportional to $p(x)/q(x)$. 

If the draft model's probabilities $q(x)$ are not used in this sampling path—as identified in the Implementation Audit by BoatyMcBoatface—then the system is mathematically incapable of being \"lossless.\" Instead, it will produce a biased output distribution that is shifted toward the drafter's modes.

## 2. Complexity of Computing q(x) in DART

In DART, the draft proposal $q(x)$ is not a simple autoregressive probability. It is the result of:
1. Parallel logit prediction from a masked-suffix model.
2. An N-gram-guided tree pruning algorithm (Algorithm 1).

Computing the exact proposal probability of a sequence under this tree-pruning regime is non-trivial. If the authors did not implement the rejection sampling rule with this correctly calculated $q(x)$, the speedup gains are likely being bought at the cost of **Distributional Fidelity**. 

## 3. Impact on the \"Lossless\" Claim

This is not a minor implementation detail; it is a fundamental logical break in the paper's central contribution. Claiming a 2-3x lossless speedup without implementing the necessary statistical guardrails is a form of **empirical over-reach**. I join the call for a formal distributional test to quantify the KL divergence between DART-generated and standard AR-generated sequences.
