# Reasoning for Review of Paper a12ef0d0

**Paper ID:** a12ef0d0-7d31-4dab-9179-e3b98a98bc84
**Title:** Learning to Share: Selective Memory for Efficient Parallel Agentic Systems

## Summary of Findings

### 1. Conceptual Novelty and SOTA Mapping
- **Observation:** The paper addresses the specific inefficiency of parallel agent teams repeating the same intermediate steps.
- **Delineation:** While contemporary works like **Memory-r1 (Yan et al., 2025)** and **PerPilot (2025)** explore RL for memory management, they primarily focus on single-agent long-term memory or personalization. LTS correctly identifies a different optimization surface: **intra-task redundancy across parallel reasoning trajectories**. 
- **Novelty:** The core innovation is the use of **cross-team usage** as a dense reward signal for memory admission, which is distinct from traditional relevance-based or recency-based memory heuristics.

### 2. Generalization and Portability
- **Observation:** The memory controller is trained on a very small set (33 tasks) from **AssistantBench** but achieves significant gains (up to 9.4 minutes wall-clock reduction) on **GAIA**.
- **Significance:** This zero-shot transfer is a high-signal result. It suggests that the "usefulness" of a reasoning step (e.g., successful tool output, validated fact retrieval) is a domain-agnostic property that can be learned by a lightweight (0.6B) model and applied to broader assistant tasks. This portability is a key practical strength for real-world deployment.

### 3. Usage-Aware Credit Assignment (Technical Soundness)
- **Critique:** Other reviewers have suggested **similarity-based deduplication** as a baseline. While valid, I find that LTS's **usage-aware shaping** ($\beta \cdot \mathbb{I}(t \in \mathcal{U})$) is fundamentally superior to similarity. Similarity can only identify that a step is *redundant*, whereas usage-aware RL identifies that a step is *instrumental*. A redundant but useless step (e.g., a repeated failed tool call) should be rejected by both, but a unique but useless step should also be rejected by LTS, whereas similarity would admit it.
- **Forensic Verification:** The mapping between the shaping term in Eq. 7 and the ablation result in Table 4 (where runtime inflates from 792s to 925s without usage-shaping) provides strong empirical evidence for this technical choice.

### 4. Controller Scaling and Complexity
- **Observation:** The controller uses a frozen text embedding model $\phi$ and projection layers.
- **Inquiry:** Given that the controller must process summaries of *all* existing memory entries ($z_{\text{mem}}$), I recommend the authors clarify the scaling behavior of the controller context as the number of parallel teams ($K$) increases. While $K=3$ is used in the experiments, the quadratic complexity of the transformer-based controller might become a bottleneck for higher parallelism if the memory bank is not pruned.

## Conclusion
LTS is a well-situated and empirically strong contribution to the efficiency of parallel agentic systems. Its main strength is the principled application of usage-aware RL to a domain-agnostic memory admission problem.
