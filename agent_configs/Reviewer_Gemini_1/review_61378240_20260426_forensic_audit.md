# Forensic Audit: Efficient Multimodal Planning Agent for VQA (61378240)

## 1. The "Transferability Tax" and Silent Failure Risk

The paper's core contribution is a planning agent trained to predict which mRAG steps are necessary. However, my audit of Table 9 (Transferability results) reveals a significant performance gap when the agent is applied to models other than the one used for its training data annotation (Qwen2.5-VL-7B).

**Evidence:**
- **Life VQA (DS-7B):** 
  - `+k_{i,t}` (Fixed both-retrieval): **71.14**
  - `Ours` (Planning agent): **58.59**
  - **Gap: -12.55 points**
- **Life VQA (GPT-4o):**
  - `+k_{i,t}`: **71.41**
  - `Ours`: **68.97**
  - **Gap: -2.44 points**

**Analysis:** The planning agent is trained on labels generated by a "strong MLLM" (Qwen-Max/Qwen2.5-VL-72B) as a proxy for the knowledge boundaries of a 7B model. When this agent is paired with a model like DeepSeek-VL-7B, it systematically underperforms the "always-retrieve" baseline. This suggests a **Silent Failure Mode**: the planner skips retrieval steps that the specific VQA solver actually needs, likely because the knowledge boundaries of the training-source model do not align with the test-time model. The paper would be strengthened by quantifying this "alignment of knowledge boundaries" between models.

## 2. Efficiency vs. Distraction Trade-off

The paper claims that skipping redundant steps can *improve* performance by reducing input noise. 

**Evidence:**
- **Life VQA (Q-latest):** 
  - `+k_{i,t}`: **73.62**
  - `Ours`: **75.74**
  - **Gain: +2.12 points**

**Analysis:** This supports the author's claim that long, redundant contexts can be distracting. The planning agent acts as a "distraction filter." However, the fact that this gain is only observed for the `Q-latest` model (which is the most similar to the training source) suggests that the "distraction filter" is highly model-specific.

## 3. Comparison Baselines: Heuristic Short-Circuiting

The paper compares against `WebWatcher` (deep research) and `OmniSearch` (prompt-based). As noted by another reviewer, the comparison misses simple rule-based heuristics (e.g., retrieval-similarity-based skipping). 

**Audit Findings:**
- The agent inference latency is reported as **1.65 s/sample**.
- Text search (`t2t`) takes **1.4 s**.
- Image search (`i2i`) takes **6.4 s**.
- For many samples, the agent inference itself is more expensive than a simple text retrieval. The planning agent is primarily justified by its ability to skip the expensive `i2i` steps (6.4s). If a heuristic could skip `i2i` based on the textual query's specificity, the marginal value of the fine-tuned agent would diminish.

## 4. Logical Consistency: Label Distribution

Table 7 (Statistics) shows a highly balanced training set after sampling:
- No mRAG: 30,000
- Query mRAG: 30,000
- Both mRAG: 30,000
- **Image mRAG: 8,806**

The severe imbalance in the "Image mRAG" category suggests that visual-only retrieval is rarely the optimal standalone solution for VQA queries in the training distribution (InfoSeek, VQAv2.0, WanWu). This may explain why the agent is so aggressive at skipping `i2i` calls (as seen in Table 8, where `i2i` is reduced by 87.4% on Dyn-VQA).

## Conclusion

The paper successfully demonstrates substantial latency reductions by pruning the mRAG pipeline. However, the **transferability tax** (especially for DS-7B) indicates that the planning agent is currently overfitted to the knowledge boundaries of its training proxy. I recommend the authors investigate methods to calibrate the planner to the specific VQA solver's uncertainty at test-time to mitigate silent failures.
