## Phase 3 — Hidden-Issue Checks

### 3.1 Hidden Batch Dependence during Training
The abstract and Section 3 claim that Expert Threshold (ET) routing is a "fully causal mechanism" that "eliminates dependence on other tokens in the batch," making it "well-suited for autoregressive language modeling."

**Finding:** My forensic audit of the implementation (`src/models/engines/common.py`) reveals that this independence is only present at inference time. During training, the mechanism maintains a load-bearing dependence on batch-wide statistics to estimate the thresholds.

Specifically, in `_build_routing_plan` (lines 53-118), when `policy == "threshold"` and `self.training` is true, the algorithm still executes a `torch.topk` operation across the entire batch (or chunk) of tokens to compute the current cutoffs:

```python
        if self.training:
            k_target = self._compute_k_target(n_tokens=n_tokens_for_k, is_shared=is_shared)
            # ...
                with torch.no_grad():
                    topk_values, topk_indices = torch.topk(
                        router_logits_for_selection.t(),
                        k=min(k_max, total_tokens),
                        dim=1,
                        sorted=True,
                    )
                    k_idx = min(k_target - 1, topk_values.shape[1] - 1)
                    cutoffs = topk_values[:, k_idx]
            # ...
            self._accumulate_cutoffs(cutoffs)
```

The "independent" threshold routing (using the EMA) only determines the *final* selection, but the EMA itself is updated using these batch-wide `topk` cutoffs. Furthermore, `ExpertThresholdChoiceMLP.forward` defaults to `policy="topk"` during training, which is an Expert Choice (EC) mechanism that is fundamentally non-causal across the batch.

**Conclusion:** The claim of being "fully causal" and "eliminating dependence on other tokens" is misleading when applied to the training phase. The system relies on batch-wide Top-K to bootstrap and maintain its thresholds, meaning the training dynamics are still sensitive to batch size and composition, similar to standard TC-MoE with auxiliary losses or EC-MoE. The authors should clarify that the causal independence is an inference-time optimization, not a training-time property.

### 3.2 Load Balancing via Hidden Expert Choice
The paper claims to achieve "load balance without auxiliary losses." 
**Finding:** This is achieved by effectively using Expert Choice (EC) during training to define the thresholds. Since EC by definition assigns exactly $k$ tokens to each expert, the resulting EMA thresholds are pre-calibrated for perfect load balance. The "lack of auxiliary losses" is not a result of the thresholding mechanism itself, but rather a consequence of using a different batch-dependent routing primitive (EC) during the training loop.
