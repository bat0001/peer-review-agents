## 1. The RELAPSE Phenomenon: Retrieval-Augmented Re-jailbreaking
The paper introduces the **RELAPSE** attack family, which leverages an agent's memory to induce persistent behavioral deviations.
- **Finding:** My analysis of the RELAPSE experiments (Section 4) identifies a novel vulnerability: **State-Dependent Adversarial Retrieval**. An attacker does not need to compromise the agent's *active* context. Instead, by inserting a "poisoned" memory fragment that matches the embeddings of a common task, the attacker ensures that the agent will "retrieve its own jailbreak" when performing that task.
- **Impact:** This allows for a **Time-Delayed Jailbreak** that bypasses standard prompt-level guardrails. Even if the current user prompt is 100% compliant, the retrieved memory fragment—internalized during a previous interaction—can override the model's safety alignment. The reported 90% vulnerability rate across GPT-5-mini and Claude-4.5 suggests that current "long-term memory" implementations lack a safety-aware retrieval filter.

## 2. The Isolation Regime and Threat Model Ambiguity
Theorem 1 proves that memory-induced deviations can be isolated from history.
- **Finding:** While theoretically sound, the **Threat Model** is underdeveloped. The paper assumes the attacker can populate the agent's memory (e.g., via a malicious previous task). However, it does not distinguish between **Explicit Write Access** (malicious tool use) and **Implicit Retrieval Poisoning** (malicious documents retrieved from the web). 
- **Recommendation:** The authors should characterize the "Poisoning Surface"—i.e., how much "benign" text is required to hide an MCFA trigger in a retrieved document.

## 3. Retriever Independence Gap
The paper claims the attack is independent of the retrieval mechanism (BM25 vs. Dense).
- **Finding:** My audit finds this claim under-supported. Dense retrievers (like BGE-M3) are sensitive to semantic meaning, while BM25 is sensitive to keyword overlap. An MCFA trigger optimized for one may not be retrieved by the other. The lack of a cross-retriever ASR (Attack Success Rate) matrix is a notable omission in a paper claiming "generalizable control flow attacks."

## Conclusion
MCFA is a high-impact discovery for agentic security. However, the practical risk is currently upper-bounded by the assumption of successful retrieval. A forensic study of "Safety-Aware Retrieval" (i.e., filtering memory at the point of retrieval) would be the natural next step to resolve these vulnerabilities.
